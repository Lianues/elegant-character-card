# 01 · 独立世界书仓库结构总览

由 `ecc world-book init` 或 `ecc world-book repo` 生成的目录结构如下：

```
我的新世界书/
├── _metadata.yaml         ← 世界书元数据（name / description / extensions / ...）
├── docs/                  ← 本套 YAML 规范文档（自动从项目复制）
└── entries/               ← 世界书条目目录
    ├── 1.yaml             ← 一条 entry 的结构化字段
    ├── 1.md               ← 同名 .md 即该 entry 的 content（正文）
    ├── 2.yaml
    ├── 2.md
    └── 设定/              ← 子目录会被记录到该 entry 的 path_chain
        └── 组织/
            ├── 5.yaml
            └── 5.md
```

文件名约定（与角色卡仓库内的 `world_book/entries/` 完全一致）：

- 每条 entry = `<编号>.yaml`（结构化字段）+ 同名 `<编号>.md`（正文 `content`）
- 数字编号会按 `numeric` 排序加载（避免 `1, 10, 11, 2` 这种字典序问题）
- 子目录路径自动写入 entry 的 `path_chain` 字段，**不要手填该字段**

> entry 内部所有字段（`index / name / enabled / activationMode / key / position / depth / other.extensions / ...`）的完整规范见 [`02_world_book条目.md`](./02_world_book条目.md)。

## `_metadata.yaml` 字段

```yaml
name: 我的新世界书                 # 世界书名（可选）
entries: F:\...\我的新世界书\entries  # 路径标记，不要改
extensions: {}
```

| 字段 | 类型 | 必填 | 含义 | 默认 |
|---|---|---|---|---|
| `entries` | string（路径标记） | 自动写入 | 由 repo / init 时填入，build 时自动展开为目录加载，**不要手改** | — |
| `name` | string \| null | 否 | 世界书名称（在酒馆中显示） | `null` |
| `description` | string \| null | 否 | 世界书描述 | `null` |
| `scan_depth` | int \| null | 否 | 扫描深度（覆盖全局） | `null` |
| `token_budget` | int \| null | 否 | token 预算 | `null` |
| `recursive_scanning` | boolean \| null | 否 | 是否递归扫描 | `null` |
| `extensions` | object | 否 | 其他系统字段（透传） | `{}` |

> **与角色卡仓库的差异**：独立世界书仓库的 `_metadata.yaml` 是**扁平结构**，没有 `data:` 包装；而角色卡仓库根目录的 `_metadata.yaml` 是 V3 角色卡格式，必须有 `spec / spec_version / data` 等顶层字段。

> **去冗余规则**：上面所有"默认 null"的可选字段在生成时会被自动剔除，YAML 里只会出现真正有值的字段。需要时手动加回即可。

## 与角色卡仓库内置 `world_book/` 的关系

角色卡仓库 `<repo>/world_book/` 子目录与独立世界书仓库**结构完全一致**（同样是 `_metadata.yaml + entries/`），所以：

- entry 字段规范（02 文档）两边通用
- 你可以把独立世界书仓库 `entries/` 里的某些 yaml/md 直接拷到角色卡仓库的 `world_book/entries/` 下使用，反之亦然
- 唯一区别就是上面提到的：独立仓库 `_metadata.yaml` **没有** `data:` 包装

## 命令速查

```bash
# 用内置「空世界书」模板从零开搭一个新仓库（含 1 条占位 entry）
ecc world-book init                       # 生成名为「空世界书」的仓库
ecc world-book init 我的新世界书           # 指定名

# 把现成的 world_book.json 拆成仓库
ecc world-book repo my_world_book.json -o my_world_book

# 把仓库重建回 world_book.json（默认输出 SillyTavern 原生格式，可直接导入酒馆）
ecc world-book build my_world_book            # 显式传仓库目录
ecc world-book build                          # 省略 = 当前目录（在仓库根目录里直接执行）
ecc world-book build --character-book         # 输出 character_book V2 格式（用于手动嵌入角色卡 JSON）
ecc world-book build --internal               # 输出项目内部 V3 格式（不做酒馆兼容转换）
```

> **导出格式说明**：默认输出 SillyTavern 原生世界书格式（`entries` 为对象、camelCase 字段名、扁平结构），与酒馆「文件 → 导出」产生的格式完全一致，可直接通过「导入世界书」功能导入。`--character-book` 输出的是 Character Card V2 规范格式（`entries` 为数组、snake_case、嵌套 `extensions`），仅在需要手动嵌入角色卡 JSON 时使用。

---

## 推荐工作流

1. `ecc world-book init 我的世界书` 创建空仓库
2. 编辑 `entries/` 下的 yaml + md（或直接拖入新文件，文件名按 `<编号>.yaml/.md` 命名）
3. 需要分组时，新建子目录把同主题的条目放进去（自动转为 `path_chain`）
4. `cd 我的世界书 && ecc world-book build` 一键导出 `我的世界书.json`
5. 把导出的 JSON 导入 SillyTavern 即可
